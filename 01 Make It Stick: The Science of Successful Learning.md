## Make It Stick: The Science of Successful Learning

**Authors:** Peter C. Brown, Henry L. Roediger III, Mark A. McDaniel

---

### Core Idea (In One Line)

> Learning is deeper and more durable when it feels difficult.

The book challenges common study habits (like rereading and highlighting) and replaces them with research-backed techniques from cognitive psychology.

---

# 🧠 Key Concepts Explained Simply

## 1️⃣ Rereading ≠ Learning

Most people:

* Reread notes
* Highlight
* Cram before exams

These **feel productive** but create *illusion of mastery*.

Why?
Because recognition ≠ recall.
You recognize the material when you see it again, but you can’t retrieve it independently.

---

## 2️⃣ Retrieval Practice (The #1 Strategy)

Instead of rereading, **force yourself to recall information from memory**.

Examples:

* Self-testing
* Flashcards
* Practice problems
* Explaining concept without notes

Why it works:

* Strengthens memory pathways
* Improves long-term retention
* Exposes knowledge gaps

Think of it like:

> Each recall attempt = strengthening a neural index pointer.

---

## 3️⃣ Spaced Practice (Spacing Effect)

Cramming works short-term.
Spacing works long-term.

Instead of:

```
Study 5 hours one night
```

Do:

```
1 hour x 5 days
```

Why?
Forgetting slightly before reviewing strengthens memory consolidation.

Memory grows stronger when it is rebuilt.

---

## 4️⃣ Interleaving (Mix, Don’t Block)

Blocked practice:

```
Solve 20 similar problems in a row
```

Interleaving:

```
Mix different problem types
```

Feels harder.
Produces better transfer and adaptability.

Example:

* Instead of 20 sorting problems,
* Mix sorting, searching, recursion, DP.

This builds pattern recognition, not memorization.

---

## 5️⃣ Desirable Difficulties

Learning improves when it feels effortful.

Examples:

* Studying in varied environments
* Testing yourself before studying
* Spacing review
* Interleaving topics

If it feels easy, you're probably not learning deeply.

---

## 6️⃣ Illusions of Learning

Things that feel effective but aren’t:

* Highlighting
* Massed practice
* Fluency from rereading
* Watching solutions instead of solving

Fluency = familiarity
Learning = retrieval + application

---

## 7️⃣ Growth Mindset & Intelligence

The book reinforces that:

* Intelligence is not fixed
* Effort + strategy changes brain structure
* Struggle is part of learning

Struggle = signal of growth.

---

# 🏗 What This Means Practically

Instead of this:

```
Read → Highlight → Reread → Feel confident
```

Do this:

```
Read → Close book → Recall → Self-test → Space → Mix topics
```

# 🎯 Big Takeaways

1. Learning should feel hard.
2. Testing is learning, not assessment.
3. Forgetting is part of strengthening memory.
4. Mix practice to build flexible thinking.
5. Confidence during study is a poor indicator of mastery.

---

# 💡 If You Apply This to Engineering

Instead of:

* Watching coding tutorials repeatedly

Do:

* Solve problems without IDE hints
* Re-implement data structures from scratch
* Review previous code after 1 week
* Mix system design + algorithms + debugging

That’s how concepts “stick.”
